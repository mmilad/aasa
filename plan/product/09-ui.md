## 9. UI — first-party operator panel

- **Stack**: **Blazor Web App**, **Interactive Server** (C# / .NET single language with the Worker).
- **Project**: separate assembly in the solution (e.g. `ArkServerManager.Admin`), **not** embedded public SaaS UI.
- **Network**: **loopback-first** (separate port from Worker acceptable, e.g. Worker `5080`, Admin `5081`); LAN/public bind only with explicit opt-in + warnings, consistent with [ADR-0003](../wiki/decisions/ADR-0003-secrets-and-exposure.md).
- **Integration**: server-side `HttpClient` to Worker **`/api/v1/...`**; header **`X-Api-Key`** from configuration / user secrets — never as a hardcoded SPA bundle secret.
- **MVP screens** (incremental): server list; server detail shell; enqueue job + poll job status; logs / INI editors follow API readiness per [08-api.md](08-api.md).

**Non-goals:** normative first-party **TypeScript** SPA; anonymous **Internet-facing** dashboard.
