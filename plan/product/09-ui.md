# Product: UI (local Next.js admin)

Normative decisions: [ADR-0005](../wiki/decisions/ADR-0005-local-admin-ui-nextjs.md), [mvp-scope.md](../wiki/requirements/mvp-scope.md) item 8.

## 9. UI — first-party operator panel

- **Stack**: **Next.js** (App Router) + Tailwind + shadcn/ui for the operator UI.
- **Project**: separate web app at repo root under **`apps/web`** (pnpm workspace package), **not** embedded public SaaS UI.
- **Network**: **loopback-first** (separate port from Worker acceptable); LAN/public bind only with explicit opt-in + warnings, consistent with [ADR-0003](../wiki/decisions/ADR-0003-secrets-and-exposure.md).
- **Integration**: Next.js server-side handlers call Worker **`/api/v1/...`** and inject header **`X-Api-Key`** from server-side configuration/user secrets (never browser-exposed).
- **MVP screens** (incremental): server list; server create form; server detail shell; enqueue job + poll job status; logs / INI editors follow API readiness per [08-api.md](08-api.md).

**Non-goals:** anonymous **Internet-facing** dashboard; browser-exposed Worker secrets.
